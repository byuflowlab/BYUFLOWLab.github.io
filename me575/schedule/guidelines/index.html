Homework Submission Guidelines

Format

All homework must be submitted in report format. A series of equations and “solutions” is insufficient. Each report should introduce the problem and any assumptions, detail the methodology, and discuss the significance and limitations of the results and conclusions. Use full sentences, not bullet points. Imagine your audience to be a colleague or supervisor—someone who understands your field but is not necessarily familiar with the details of your particular problem. Your report should not only be technically correct, but should also be written clearly and follow best practices in technical writing.

Each report has a page limit of 10 pages. A title page is not necessary, but be sure to write your name on the first page. I recommend using LaTeX to produce your reports (more on this below), but you are also welcome to use Microsoft Word and MS Equation Editor or any other typesetting/word-processing program that you prefer.

Why a report format? You will learn deeper by having to explain in words, it will better prepare you for the type of work you will have in an engineering career, and the hard work you did will be much more accessible to you when referring to it later.

You will also need to submit any source code that you developed. You should not include source code directly in your report, but you may refer to it from your report if necessary. Source code should be commented so that it is easily readable.

Submission Instructions

Each submission should consist of one PDF file for the report and one zipped directory containing any auxiliary material (e.g., source code). On Learning Suite you can upload multiple items, just upload them one right after the other. Nothing is overwritten even though it only shows you your last submitted file. If you are really worried about it, you can upload everything in one zip, but it makes it easier for us to grade if the PDF is separate.

LaTeX

LaTeX is widely used in scientific writing and if you are doing graduate work you will likely need to learn it at some point (check with your advisor, some research groups prefer MS Word). To be clear, there is nothing wrong with using Microsoft Word (or an equivalent) for your report preparation. These are just tools, what’s more important is doing good work.

If you want to use LateX, Overleaf is a good place to start unless you already have a workflow you are comfortable with. It is like Google Docs for LaTeX. There isn’t anything to install, and multiple people can work collaboratively (nice for your project). It also provides a lot of starter templates, in fact I’ve created a simple overleaf example for you that illustrates most of the commands you will use. Many LaTeX tutorials/guides exist online if you need to do anything fancier.
